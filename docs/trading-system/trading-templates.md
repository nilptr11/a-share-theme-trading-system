# 交易执行模板

本文档是 [交易模式](trading-reference.md) 的日常执行模板，用于盘前、盘中、复盘的实际记录。

---

## 盘前

盘前不预测，只做预案。盘前只回答四个问题：

```text
1. 今日外部环境偏利多、中性还是利空？
2. 今日候选主线不超过 3 个，分别是什么？
3. 每个候选主线里，核心观察股是谁？
4. 什么情况下今天不交易？
```

信息源只用于发现变量，不用于堆积信息：

```text
同一主题跨平台重复，只记录一次。
重复信息不增加权重。
盘前信息筛选不超过 20 分钟，超时后只保留最重要的 3 条影响因素。
先用一个主信息源建立框架，再用其他信息源查漏，不逐条重复阅读。
看完信息必须转化为候选主线、观察股、交易权限和禁止事项。
不能转化为交易假设的信息，忽略。
```

盘前没有进入观察池的方向，盘中即使异动，也只观察不追买。

## 盘中

盘中只按这个顺序处理：

```text
1. 先处理持仓风险。
2. 再确认市场是否允许交易。
3. 再验证主线是否被资金确认。
4. 最后等待核心股标准买点。
```

盘中不追异动，只验证主线。

### 盘中检查

```text
持仓：是否触发止损、风险预算亏损、板块退潮或市场恶化？
市场：市场评分是否 ≥ 6？上涨家数和成交额是否支持交易？
主线：能否写清主线名称、资金确认和核心股？
个股：是否属于主线、强于板块、出现标准买点、止损位清楚？
```

主线写不清楚时，不开新仓。

### 上涨家数校准

上涨家数只校准市场环境，不构成买入信号：

```text
上涨家数 < 800：情绪冰点，不开新仓或只观察。
上涨家数 800-2000：弱势分化，只在市场评分 ≥ 6 且买点全部满足时轻仓试错。
上涨家数 2000-3500：正常环境，可按市场评分执行。
上涨家数 > 3500：情绪过热，不追高，优先等分歧回踩。
```

### 盘中禁止

```text
异动提醒不是买点。
热榜不是买点。
条件选股不是买点。
无板块联动的单股拉升不买。
10:30 前主线不清楚，不急于开仓。
```

## 复盘

复盘顺序：先自己复盘盘面，再看外部解析做补充验证，最后形成次日计划。

复盘不只解盘，必须生成次日计划。

### 复盘边界

```text
复盘只反馈判断质量和执行纪律。
复盘结论不能在盘中临时放宽规则。
要改变买点、卖点、仓位或暂停机制，必须先修改交易模式文档。
每月只检查三件事：亏损交易是否违反规则，连续符合规则的交易是否仍然亏损，核心数值是否明显不适应当前市场。
连续符合规则仍亏损时，先降低风险预算或暂停观察，不盘中临时改规则。
```
