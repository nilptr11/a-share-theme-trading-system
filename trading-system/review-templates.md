# 复盘模板

> 用于每日、每周和定期复盘。本模板只记录执行结果和反馈结论，不定义交易规则。

全市场成交额（亿）：____  涨停/跌停（排除ST）：____/____

## 2. 每周复盘（差异项）

在每日复盘基础上，汇总：

```text
本周交易次数 / 胜率 / 盈亏比 / 最大单笔盈亏 / 周盈亏合计（扣成本）
最大回撤 / 最赚钱买点类型 / 最亏钱买点类型
是否有市场评分 < 6 的实盘交易
是否有状态 B 使用 6.3 的违规交易
是否有风险预算超过规则上限的交易
是否有未估算交易成本的交易
是否有冲动交易 / 违反止损 / 过度交易 → 下周改进重点
```

## 3. 定期复盘分析

```text
1. 三种买点类型的盈亏表现对比
2. 状态 A vs 状态 B 下的交易表现
3. 不同市场评分（6-7 vs 8-10）下的交易表现
4. 不同仓位下的盈亏分布
5. 不同时段表现（早盘 vs 尾盘，周一 vs 周五）
6. 最大盈利和最大亏损交易的共同特征
7. 违反规则的交易占比及盈亏
8. 交易成本合计，占毛盈利的比例
9. 扣成本后净盈亏、胜率、盈亏比、最大回撤
10. 超风险预算交易次数
```

复盘回答三个问题：

```text
1. 有没有按系统交易？
2. 亏损是规则内试错，还是违反规则？
3. 如果要改变交易动作，是否已经先修改 trading-reference.md？
```

复盘结论只作为执行反馈；买点、卖点、仓位和暂停机制仍以 `trading-reference.md` 为准。
